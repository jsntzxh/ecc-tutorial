## 第二部分：核心概念（1:30 - 5:00）

### 画面
展示 ECC 架构图，五大核心组件：Rules → Skills → Agents → Commands → Hooks

### 旁白
首先，让我们了解 ECC 的五大核心组件。

**第一个是 Rules——规则。**
规则定义了 AI 在编写代码时应该遵循的标准。ECC 提供了 89 条规则，覆盖 14 种编程语言。每种语言都有 5 个核心规则文件：编码风格、设计模式、安全规范、测试规范和钩子使用。当你写代码时，Claude Code 会自动加载对应的规则，确保代码质量始终如一。

**第二个是 Skills——技能。**
技能是封装好的专业工作流，包含 147 个。比如你想做代码审查，就调用 code-review 技能；想做安全扫描，就调用 security-scan 技能。每个技能都包含了该领域的最佳实践。

**第三个是 Agents——智能体。**
ECC 有 48 个专业智能体，它们是能自主执行复杂任务的 AI 代理。比如 TypeScript 项目就用 typescript-reviewer 智能体来审查代码，Java 项目就用 java-reviewer。还有专门的智能体处理构建错误、性能优化、数据库设计等。

**第四个是 Commands——命令。**
68 条斜杠命令让你用最简短的输入触发最复杂的操作。输入 `/code-review` 就会自动审查代码，输入 `/plan` 就会启动架构设计流程。

**第五个是 Hooks——钩子。**
钩子是自动化机制，能在特定事件发生时自动执行。比如在 Bash 命令执行前进行安全验证，或在会话结束时自动保存状态。

## 第三部分：安装演示（5:00 - 7:30）

### 画面
展示终端窗口，逐步演示安装过程

### 旁白
现在让我们来安装 ECC。安装非常简单，有两种方式。

**方式一：通过 npm 安装。**
打开终端，输入 `npm install -g ecc-universal`。安装完成后，运行 `npx ecc-install --profile full` 安装完整配置。如果你想只安装特定语言的配置，可以用 `npx ecc typescript` 或 `npx ecc python`。

**方式二：通过 Claude Code 插件系统安装。**
在 Claude Code 中直接输入 `/plugin install everything-claude-code@everything-claude-code`，系统会自动下载和配置插件。

安装完成后，可以验证一下：查看 `~/.claude/rules/ecc/` 目录应该能看到所有规则文件，查看 `~/.claude/skills/ecc/` 应该看到 147 个技能目录。

## 第四部分：规则系统详解（7:30 - 10:00）

### 画面
展示规则目录结构和示例代码

### 旁白
让我们深入了解规则系统。ECC 的规则分为三个层级：

**通用规则** 位于 `common` 目录下，包含 10 个核心文件，适用于所有项目。比如 `security.md` 定义了 OWASP Top 10 的安全防护要求，`coding-style.md` 定义了命名规范和代码格式。

**语言专用规则** 每种语言有自己专属的规则目录。当你打开一个 Go 项目，ECC 会自动加载 Go 的规则；打开 Python 项目，就加载 Python 的规则。

**中文规则** 位于 `zh` 目录，是通用规则的中文翻译版，方便中文开发者阅读。

语言规则会覆盖通用规则中的对应条目。你也可以在项目中创建 `.claude/rules/` 目录来添加项目级规则，实现最精细的控制。

举个例子，当你在写 Java 代码时，ECC 的 Java 规则会确保你遵循分层架构、正确的 JPA 使用方式、以及 Spring Boot 的安全配置。

## 第五部分：技能系统实战（10:00 - 13:00）

### 画面
展示在 Claude Code 中调用技能的实际效果

### 旁白
现在来看技能系统如何在实际开发中使用。

调用技能非常简单，用斜杠加技能名即可。比如写完了代码想审查一下，输入 `/code-review`，ECC 会自动启动代码审查智能体，按照对应语言的规范逐项检查。

这里给大家列出最常用的几个技能：

- `code-review` —— 代码修改后的必做检查
- `build-fix` —— 编译出错时自动诊断和修复
- `test-coverage` —— 确保你的测试覆盖率达标
- `plan` —— 开始新功能前先做好架构设计
- `security-review` —— 涉及用户输入或认证时必做的安全检查
- `review-pr` —— 提交 Pull Request 前的最后一道关卡

对于各语言开发者，ECC 还有专属技能。比如 Go 开发者有 `go-review`、`go-build`、`go-test`；Rust 开发者有 `rust-review`、`rust-build`、`rust-test`。

## 第六部分：智能体深度解析（13:00 - 16:00）

### 画面
展示智能体分类列表和调用示例

### 旁白
智能体是 ECC 最强大的功能之一。让我给大家介绍几个最重要的：

**代码审查智能体。** ECC 为每种语言都配备了专属的审查智能体。TypeScript 的智能体会检查类型安全、异步正确性和 Node/Web 安全。Rust 的智能体会检查所有权、生命周期和 unsafe 代码的使用。Java 的智能体会检查 Spring Boot 的分层架构和 JPA 模式。

**构建修复智能体。** 当构建失败时，不要慌张。ECC 有专门的智能体来修复各语言的构建错误——Go 的 vet 问题、Rust 的 borrow checker、C++ 的模板错误，都能自动诊断和修复。

**安全智能体。** security-reviewer 会检测 OWASP Top 10 漏洞，silent-failure-hunter 会找出那些被吞掉的错误和静默失败。

**开源发布流水线。** ECC 甚至提供了完整的开源发布工具链——forker 负责复制代码并剥离敏感信息，sanitizer 负责验证安全性，packager 负责生成完整的开源项目文件。

## 第七部分：推荐工作流（16:00 - 18:30）

### 画面
展示工作流流程图

### 旁白
现在让我给大家展示几个推荐的开发工作流：

**新功能开发流程：**
先用 `/plan` 做好架构规划，然后用 `/feature-dev` 实现功能，完成后用 `/code-review` 审查代码，最后用 `/test-coverage` 确保测试覆盖。

**Bug 修复流程：**
描述 Bug 现象让 Claude Code 定位问题，实施修复后运行 `/build-fix` 确保编译通过，然后用 `/code-review` 审查修复质量，最后用 `/test-coverage` 补充测试。

**PR 提交流程：**
ECC 提供了完整的 PR 流水线——`/prp-plan` 创建计划，`/prp-implement` 实现修改，`/prp-commit` 生成规范的提交信息，`/prp-pr` 创建 Pull Request 并生成描述。

**持续集成：**
你还可以设置定时任务，比如用 `/loop-start 10m /build-fix` 让 ECC 每 10 分钟自动检查构建状态。
